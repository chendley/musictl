# musictl - Project Context for Claude

## Project Overview

**musictl** is a CLI toolkit for managing, fixing, and organizing music libraries. Built for personal use - focused on safety, correctness, and no external dependencies.

**Core Principle:** Safety first. Dry-run by default, explicit `--apply` required for destructive operations.

**License:** GPLv3 (prevents commercial exploitation)

**Current Version:** 0.1.0 (initial release)

---

## Critical Background - Encoding Bug Incident

**What happened:** The initial encoding detection was fundamentally broken. It encoded mutagen's display strings as latin-1 instead of reading raw bytes, causing it to flag CORRECT files as broken and corrupt them during "fixing". Corrupted 1,139 files before emergency reversal recovered 910.

**The fix:** Rewrote `detect_non_utf8_tags()` with dual-pattern detection:
1. Pattern 1: UTF-8 mojibake (double-encoding)
2. Pattern 2: Wrong encoding header (CP1251 shown as Latin-1)

**Safety checks:** Only flag files where reversal produces FEWER anomalous characters. Count characters outside expected Unicode ranges (basic Cyrillic U+0410-0x044F, Japanese Hiragana/Katakana/CJK, Chinese CJK).

**Testing requirements:** MUST test encoding operations on files with CORRECT encoding (should skip them) and actual mojibake (should fix them).

### Key Dependencies
- **mutagen** - Audio metadata (MP3, FLAC, OGG, M4A, etc.)
- **typer** - CLI framework
- **rich** - Terminal formatting (progress bars, tables, colors)
- **ffprobe** (optional) - Advanced audio analysis (sample rate, bit depth)

### Design Patterns

**Command structure:**
- Every command module exports a `typer.Typer()` app
- Main CLI imports and mounts them as subcommands
- Commands follow pattern: validate input → process files → display results
- Always support `--recursive` and `--apply` flags

**Safety pattern:**
- Default: dry-run (preview only)
- `--apply` flag required to make changes
- Show before/after preview for modifications
- Check for data loss scenarios (e.g., strip-v1 checks for ID3v2)

**Error handling:**
- Graceful degradation - continue processing on individual file errors
- Clear error messages with file paths
- Use `typer.Exit(code)` for clean exits
- Support Ctrl+C cancellation

## Development Principles

### Code Style

1. **Safety over convenience** - Always dry-run by default
2. **No magic** - Explicit is better than implicit
3. **Test everything** - Especially destructive operations
4. **Clean abstractions** - Extract helpers for repeated logic
5. **No flowery language** - Technical, straightforward documentation

### Anti-Patterns to Avoid

❌ **Don't:**
- Add emojis to code output (only in tables for safety indicators)
- Use marketing language ("powerful", "beautiful", "production-ready")
- Auto-apply destructive operations
- Skip safety checks for convenience
- Add external API dependencies (MusicBrainz, AcoustID) without discussion
- Modify files without preview

✅ **Do:**
- Extract repeated logic to helpers (see `format_size()`, `_count_anomalous_chars()`)
- Optimize I/O (check multiple things in one file open)
- Write comprehensive tests
- Document safety considerations
- Preserve encoding-specific checks (Cyrillic, Japanese, Chinese)

### Testing Requirements

**Critical test cases:**
1. Dry-run vs apply mode for all destructive operations
2. Encoding operations on CORRECT files (must skip)
3. Encoding operations on mojibake (must fix)
4. ID3v1 strip safety (skip v1-only files unless --migrate/--force)
5. File I/O errors (permissions, missing files)

**Test coverage:** 167 tests, all must pass before commit.

**Run tests:**
```bash
uv run pytest              # All tests
uv run pytest -xvs        # Stop on first failure, verbose
```

## Common Tasks

### Adding a New Command

1. Create command module in `src/musictl/commands/`
2. Define `app = typer.Typer(help="...")`
3. Implement command functions with `@app.command()` decorator
4. Import and mount in `src/musictl/cli.py`
5. Write tests in `tests/test_<command>.py`
6. Update README command reference

**Example:**
```python
# src/musictl/commands/newcmd.py
import typer
from musictl.utils.console import console

app = typer.Typer(help="New command operations")

@app.command()
def subcmd(
    path: Path = typer.Argument(..., help="Path to process"),
    apply: bool = typer.Option(False, "--apply", help="Apply changes"),
):
    """Description of what this does."""
    target = Path(path).expanduser().resolve()
    # ... implementation
```

### Adding New Utility Functions

Extract repeated logic to `src/musictl/utils/console.py` or create new utility modules as needed. Examples:
- `format_size()` - Size formatting (already exists)
- `_count_anomalous_chars()` - Character counting (in encoding.py)
- `_parse_id3v1_field()` - ID3v1 parsing (in tags.py)

### Encoding-Related Changes

**CRITICAL:** When modifying encoding detection or fixing:
1. Preserve all encoding-specific character checks (Cyrillic, Japanese, Chinese)
2. Test on CORRECT files first (must not be flagged)
3. Test on actual mojibake (must be fixed)
4. Verify reversal produces MORE valid characters for target script
5. Add tests to `tests/test_encoding_safety.py`

### Performance Optimizations

**Before optimizing:**
- Profile first - don't guess
- Most operations are I/O bound (file reads)
- Focus on reducing file opens, not micro-optimizations

**Recent optimizations:**
- `strip_v1`: Check ID3v1 and ID3v2 in single file open (was 2 opens)
- `clean.py`: One `rglob("*")` pass instead of 10 separate globs
- Size formatting: Extract to utility to avoid duplication

## Known Issues / Gotchas

### mutagen Behavior
- **Auto-conversion:** mutagen auto-converts ID3v1 to ID3v2 frames when reading
  - Can't detect v1-only files by checking `len(id3.keys())`
  - Must check for "ID3" header bytes at file start
- **Encoding byte:** Mutagen respects the encoding byte in ID3 frames
  - Encoding 0 = Latin-1, 3 = UTF-8
  - Wrong encoding byte causes display issues

### File I/O
- **Atomic operations:** mutagen handles tag writes safely
- **TOCTOU risk:** Don't run concurrent operations on same files
- **Permissions:** Check write permissions before modifications

### Cross-platform
- **Path handling:** Use `Path().expanduser().resolve()`
- **Line endings:** Not an issue for binary files (audio), but test text exports
- **Temp files:** Platform-specific patterns in `clean.py`
